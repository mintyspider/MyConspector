## Платформа Аврора
Корни Авроры уходят в другие операционные системы, не андроид, так что аврора - это линукс, апк запустить нельзя, но зато можно запускать rpm =)

Открытая мобильная платформа.

В основном корпоративный сегмент (Центр управления устройствами (вплоть до 20к), Маркет (ограниченный набор приложений для корпоративного использования), Пуш сервис (локально в конкретной компании))

Специальный графический интерфейс для мобильных устройств.

> “самое красивое приложение на Авроре - календарик”)

- Как опубликовать приложение:
Подтвержденный вк акк — в яндекс браузере консоль рустора, там в интерфейсе история про загрузку, ключи для подписи пакета (2 недели), через консоль/сдк по инструкции на портале подписать пакет rpm, проверить валидацию (запись на устройстве), далее загрузить на рустор сопутствующие артефакты (иконка, скриншоты, описание), потом опубликовать, ждать 1-2 дня модерацию, повторная публикация (подписать, валидировать, опубликовать, дождаться модерацию)

Аврора заточена только под мобилки с сенсорными небольшими экранами. СДК есть только у Авроры =)), еще есть эмулятор операционной системы прямо на пк.

## Stateless vs Stateful

1. Stateless: неизменяемые виджеты (после отрисовки не изменяются)
2. Высокая производительность и малый вес
3. Stateful: имеет объект State, который живет дольше виджета
4. Метод setState(): уведомляет фреймворк о необходимости обновления
5. ЖЦ (Lifecycle): от initState до dispose
6. Использование : формы ввода, таймеры переключатели
7. Раздреление логики и предсатвления внутри Stateful

## Event Loop и асинхронность
- Однопоточность: Dart работает в одном потоке (Isolate), что исключает конфликты памяти (race conditions)
- Event Loop: бесконечный цикл, который по очереди берет задачи из очередей и выполняет их
- Event Queue: очередь события
- Microtask Queue: внутренние критические задачи Dart
- Асинхронность (Future): способ поставить задачу в очередь, не замораживая интерфейс
- Isolates: решение для тяжелых вычислений (аналог thread, но не он, нет общей памяти)
- Плавность: Event Loop позволяет обрабатывать ввод пользователя между кадрами и отрисовки

3. Widget Inspector: возможность ткнуть в экран и найти виджет в коде
