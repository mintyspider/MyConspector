# 32. Для чего и как применяются метрики объектно-ориентрованных систем?	

## Зачем применяются метрики ОО-систем
1. Оценка качества архитектуры - раннее выявление проблем (несвязность, хрупкость, высокое зацепление).
2. Прогнозирование трудозатрат - cложные классы требуют больше тестов и времени на сопровождение.
3. Контроль рефакторинга - сравнение метрик «до» и «после».
4. Обнаружение «запахов кода» (code smells) - Бог-объект, ленивый класс, цепочки вызовов, высокое сцепление.
5. Бенчмаркинг - сравнение проектов или версий системы.
6. Назначение приоритетов ревью - сначала проверяются классы с «плохими» метриками.

## Процесс применения метрик
1. Выбрать метрики (обычно: LCOM, CBO, DIT, WMC, RFC, NOC)
2. Установить пороговые значения (например, CBO < 10, DIT < 5)
3. Подсчитать автоматически (скрипт / CI / инструмент)
4. Интерпретировать – не всегда большое значение = проблема
5. Провести рефакторинг классов, выходящих за пороги
6. Пересчитать метрики – проверить улучшение

- Для оценки качества разработанных моделей используется специальный метрический аппарат

- Для объектно-ориентированных систем выделяют пять характеристик качества: 
  - локализация, 
  - инкапсуляция, 
  - информационная закрытость, 
  - наследование, 
  - абстракция
