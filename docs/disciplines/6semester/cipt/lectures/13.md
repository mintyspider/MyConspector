# 13. Криптоанализ со стороны атакующего. Задача дискретного логарифмирования.

## Криптоанализ

Криптоанализ - наука восстановления открытого текста или ключа без легального доступа к нему, а также выявление уязвимостей алгоритмов.

### Цели криптоанализа

- получение информации о сообщении;
- восстановление ключа;
- снижение криптостойкости алгоритма;
- оценка надежности криптосистем.

### Стратегия криптоанализа

Основная идея криптоанализа заключается в поиске математических закономерностей и слабых мест алгоритма.

Обычно анализ проводится:
- на упрощённых или усечённых версиях алгоритма;
- на небольшом количестве раундов;
- с использованием статистических зависимостей.

Шифр считается уязвимым, если найден способ атаки быстрее полного перебора ключей.

Полный перебор — проверка всех возможных ключей до нахождения правильного.

### Модели атак криптоанализа

#### Атака только по шифртексту (Ciphertext-only attack)

У атакующего есть только зашифрованные сообщения.

Цель:
- восстановить открытый текст;
- определить ключ.

#### Атака с известным открытым текстом (Known-plaintext attack)

Известны пары:
- открытый текст;
- соответствующий шифртекст.

Используется для поиска ключа или структуры шифра.

#### Атака с выбранным открытым текстом (Chosen-plaintext attack)

Атакующий может самостоятельно выбирать открытые тексты и получать соответствующие шифртексты.

Очень опасная модель для многих алгоритмов.

#### Атака с выбранным шифртекстом (Chosen-ciphertext attack)

Атакующий выбирает шифртекст и получает результат расшифрования.

Используется против асимметричных криптосистем.

### Классификация исходов криптоанализа блочных шифров

#### Информационная дедукция

Получение косвенной информации:
- формат документа;
- язык текста;
- наличие стандартных слов;
- структура файла;
- отдельные биты ключа.

#### Частичная дедукция

Расшифровка части сообщений или отдельных блоков данных при определённых условиях.

#### Глобальная дедукция

Получение возможности расшифровывать сообщения без знания исходного ключа.

Фактически создаётся эквивалент алгоритма дешифрования.

####Полный взлом

Полное восстановление секретного ключа.

Последствия:
- чтение всех прошлых и будущих сообщений;
- подделка сообщений;
- нарушение аутентичности и конфиденциальности.

## Задача дискретного логарифмирования

### Определение

Задача дискретного логарифмирования — одна из основных сложных математических задач, используемых в криптографии.

Пусть:
- $p$ — простое число;
- $g$ — первообразный корень по модулю $p$;

известно значение:

$$
y \equiv g^x \pmod p
$$

Требуется найти показатель степени $x$.

Число $x$ называется дискретным логарифмом числа $y$ по основанию $g$.

### Почему задача считается сложной

Вычислить:

$$
y = g^x \mod p
$$

легко даже для больших чисел.

Обратная задача:
- найти $x$ по $y$ - вычислительно очень сложна.

На этой сложности основана безопасность многих криптосистем.

### Применение задачи дискретного логарифмирования

Используется в:
- алгоритме Диффи — Хеллмана;
- цифровой подписи DSA;
- алгоритме Эль-Гамаля;
- криптографии на эллиптических кривых.
