# 14. Криптосистема Диффи-Хеллмана.

## Описание криптосистемы

Год создания: 1976

Создатели:
- Уитфилд Диффи
- Мартин Хеллман

Цель криптографического протокола Диффи-Хеллмана - создание двумя абонентами, использующими открытые каналы передачи данных, общего секретного ключа.

Криптостойкость протокола Диффи-Хеллмана основана на сложности решения задачи дискретного логарифмирования - поиска значения числа x, для которого $g^x\ mod\ p = X$.

> Классический DH не защищён от атаки «человек посередине» (если каналы не аутентифицированы).

**Односторонняя функция** - математическая функция, которая *легко* вычисляется для любого входного значения, но *трудно* найти аргумент по заданному значению функции.

> Здесь "легко" и "трудно" должны пониматься с точки зрения теории сложности вычислений.

Так, для $f(x) = g^x\ mod\ p$ можно быстро вычисить $f(x)$, но для заданного y трудно (долго) найти x, для которого $y=f(x)$. 

## Параметры протокола
- $p\ -$ большое простое число;
- $g\ -$ генератор достаточно большой подгруппы в $Z^*_p$.

> [!Note] Замечание 📓
> Строго говоря, g — порождающий элемент циклической подгруппы простого порядка q, которая делит p−1.
> На практике часто берут g порядка q, где q — большое простое (не обязательно весь $Z_p^∗$). Это защита от некоторых атак.

## Шаги протокола
1. 

Алиса выбирает случайное большое число a, вычисляет и отправляет Бобу
$$X = g^a\ mod\ p$$

Боб выбирает случайное большое число b, вычисляет и отправляет Алисе
$$Y = g^b\ mod\ p$$
-----
2. 
 
Алиса вычисляет ключ $K_A = Y^a\ mod\ p$.

Боб вычисляет ключ $K_B = X^b\ mod\ p$.

----
3. 
 
В итоге формируется общий секретный ключ:

$$K_A = Y^a\ mod\ p=(g^b)^a\ mod\ p=g^{ba}\ mod\ p,$$

$$K_B = X^b\ mod\ p = (g^a)^b\ mod\ p=g^{ab}\ mod\ p,$$

Видно, что $K_A = K_B.$
