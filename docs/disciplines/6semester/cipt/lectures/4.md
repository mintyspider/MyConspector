# 4. Генерация больших псевдослучайных простых чисел

## Зачем это нужно

В асимметричной криптографии (RSA, DSA, Diffie–Hellman и др.) требуются **большие простые числа** длиной от 512 до 4096 бит. Они нужны для:
- создания криптомодуля $n = p \cdot q$ (RSA);
- выбора простого модуля $p$ (DH).

Получить такое число просто «в уме» или подбором невозможно — используются специальные алгоритмы.

## Два этапа генерации

Процесс всегда состоит из двух шагов:

1. **Генерация кандидата** — случайного нечётного числа нужной длины.
2. **Проверка на простоту** — тест, который определяет, является ли число простым.

> ⚠️ Классические детерминированные тесты (перебор делителей) для больших чисел невозможны — слишком много вариантов. Поэтому используются **вероятностные тесты**.

## Генерация случайного кандидата

### Требования:
- Число должно быть **случайным** (или псевдослучайным), чтобы злоумышленник не мог предсказать его значение.
- Длина — строго заданная (например, 1024 бита).
- Число должно быть **нечётным** (иначе оно точно не простое, кроме числа 2).

### Типичный алгоритм:
1. Берется криптостойкий генератор псевдослучайных чисел (CSPRNG) — например, на основе AES или хэш-функций.
2. Генерируется битовая строка нужной длины.
3. Устанавливаются старший и младший биты в 1 (чтобы гарантировать нечётность и заданную длину).
4. Полученное число используется как кандидат.

## Вероятностные тесты на простоту

Ни один вероятностный тест не даёт **100% гарантии**, что число простое. Однако при правильных параметрах вероятность ошибки можно сделать пренебрежимо малой ($2^{-80}$ и меньше).

### 1. Тест Ферма (теорема Ферма)

**Принцип:** Если $p$ простое, то для любого $a$, взаимно простого с $p$, выполняется:

$$a^{p-1} \equiv 1 \pmod{p}$$

**Алгоритм:**  
Выбираем случайное $a$ (основание) и проверяем равенство. Если не выполняется — число составное. Если выполняется — число **вероятно** простое.

**Недостаток:** Существуют **псевдопростые числа Ферма** (числа Кармайкла), которые проходят тест для всех $a$, но являются составными.

### 2. Тест Миллера–Рабина (основной)

**Принцип:** Более сильный тест, основанный на свойствах квадратичных вычетов и разложении $p-1 = d \cdot 2^s$.

**Почему он лучше:**
- Нет аналога чисел Кармайкла.
- Для случайного составного числа вероятность ошибочно назвать его простым — не более $1/4$ за одну итерацию.
- После $t$ итераций вероятность ошибки $\le 4^{-t}$. Для $t = 40$ это практически ноль.

**Использование:**  
Стандарт де-факто для генерации простых чисел (в OpenSSL, OpenPGP, библиотеках).

## Детерминированные тесты (для небольших чисел)

Для чисел до определённой границы существуют детерминированные проверки:

- **Перебор делителей до $\sqrt{n}$** — только для очень маленьких чисел.
- **Тесты на основе предвычисленных наборов оснований** — например, для 32-битных чисел достаточно проверить основания $\{2, 3, 5, 7, 11\}$.

Для криптографических длин (1024+ бит) детерминированные тесты **неприменимы** по времени.

## Ускорение генерации: решето и предварительная проверка

Перед запуском дорогого теста Миллера–Рабина кандидат проверяют **малыми простыми делителями**:

1. Делим число на простые числа до 100, 1000 или 10000 ($2, 3, 5, 7, 11, \dots$).
2. Если делится на любое из них — отбрасываем и генерируем новый кандидат.

Это позволяет отсеять ≈90% составных чисел очень быстро.

## Практический алгоритм генерации (стандартный)

1. **Генерация кандидата** (случайное нечётное число нужной длины).
2. **Проверка на малые делители** (до 1000–5000).
3. **Тест Миллера–Рабина** с 40–50 итерациями для криптографических длин.
4. Если тест пройден — число принимается как простое.
5. Если нет — увеличиваем кандидат на 2 (переходим к следующему нечётному) и повторяем с шага 2.

> Иногда вместо инкремента генерируют совершенно новый кандидат для улучшения случайности.

## Особые требования для криптографии

Не любое простое число подходит для RSA или DH:

- **Сильные простые числа** (strong primes) — $p$ такое, что $p-1$ и $p+1$ имеют большие простые делители. Требуются в некоторых стандартах.
- Для **DH** часто требуются простые числа с известным порядком (например, простое число, где $p = 2q + 1$, а $q$ — тоже простое; такое $p$ называется **безопасным простым**).
