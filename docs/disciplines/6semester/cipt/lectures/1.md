# 1. Симметричные шифры. AES.

## Симметричные шифры

![](../images/img1-1.png)

> Один общий ключ для шифрования и дешифрования.

::: info ℹ️ **Симметричное шифрование** — это метод шифрования данных, при котором один и тот же криптографический ключ используется и для шифрования, и для расшифровки информации.
:::

### Принцип работы
Схема работы симметричного шифрования:

1. Отправитель и получатель заранее договариваются о ключе и алгоритме шифрования.
2. Отправитель использует ключ для шифрования данных по выбранному алгоритму.
3. Зашифрованные данные (шифротекст) передаются получателю по каналу связи.
4. Получатель применяет тот же ключ и алгоритм для расшифровки сообщения.

**Ключевая проблема:** безопасная передача ключа. Если злоумышленник перехватит единственный ключ во врема обмена, вся система скомпрометирована.

### Основные требования к современным симметричным шифрам
Чтобы алгоритм считался надёжным, он должен:
- Устранять статистические закономерности исходных данных (частота символов, типичные сочетания букв и т. п. не должны прослеживаться в шифротексте).
- Быть нелинейным (не допускать выявления закономерностей даже при наличии нескольких пар «исходный текст — шифротекст»).
- Обеспечивать «эффект лавины» — при малейшем изменении исходных данных или ключа шифротекст должен меняться кардинально.

### Типы алгоритмов симметричного шифрования
1. **Блочные алгоритмы** — обрабатывают данные блоками фиксированной длины (64, 128 бит и т. д.):

> Если последний блок меньше требуемого размера, он дополняется специальными символами.

Примеры: [AES](./1.md#aes), RC5, Blowfish, Twofish.

2. **Потоковые алгоритмы** — шифруют данные побитно или побайтно с использованием гаммирования:

> Каждый бит/байт исходного текста комбинируется с соответствующим битом/байтом псевдослучайной последовательности (гаммы) через операцию XOR (⊕).

Гамма генерируется на основе ключа.

Примеры: RC4, Salsa20, HC-256, WAKE.

### Как достигается надёжность
Современные алгоритмы используют раунды — последовательность операций, повторяемых многократно:

- Подстановка (S‑блоки) — замена фрагментов данных (например, байтов) по таблице соответствий. Привносит нелинейность.

- Перестановка (P‑блоки) — перемешивание битов/байтов по заданному правилу. Способствует «эффекту лавины».

Ключевое добавление — комбинирование данных с частью ключа (часто через XOR).

#### Достоинства и недостатки симметричного шифрования

|Достоинства|Недостатки|
|---|---|
|Высокая скорость обработки данных — алгоритмы оптимизированы для быстрых математических операций|Проблема распределения ключей — ключ нужно безопасно передать получателю до начала обмена сообщениями. При передаче по незащищённому каналу его могут перехватить|
|Меньшие вычислительные затраты по сравнению с асимметричными методами|Сложность управления ключами в больших сетях — для каждой пары пользователей нужен уникальный ключ.|
|Относительная устойчивость к потенциальным атакам с использованием квантовых компьютеров|Непригодность для цифровых подписей — поскольку ключ известен обеим сторонам, нельзя доказать авторство сообщения.|

#### Области применения
Симметричное шифрование широко используется в:

- Мессенджерах (защита переписки).
- Видеоконференцсвязи (шифрование аудио‑ и видеопотоков).
- Банковских операциях (защита транзакций).
- Протоколах защищённой передачи данных (TLS/SSL — для конфиденциальности трафика после установки соединения).

## AES (Advanced Encryption Standard)

::: info ℹ️ **AES** (Advanced Encryption Standard) – усовершенствованный стандарт шифрования США.
:::

В 1997 году National Institute of Standards and Technology (NIST, Национальный институт стандартов и технологий США) объявил конкурс для выбора нового криптографического стандарта, который должен был стать преемником DES.

Требования к кандидатам:
- открытая публикация;
- ключи размером 128, 192 и 256 бит;
- предназначен для аппаратной и программной реализации;
- не должен быть запатентован;
- должен быть стойким, эффективным по времени и памяти, гибким.

В результате конкурса в 2000 году победителем был объявлен [шифр Rijndael](./1.md#алгоритм-rijndael).



Создатели: Винсент Рэймен, Йоан Даймен (Бельгия)

Создан в 1998 г.

Размер ключа: 128 / 192 / 256 бит

Размер блока: 128 бит

Число раундов: 10 / 12 / 14 (зависит от размера ключа)

Операции: подстановки, перестановки, XOR

[Более подробно тут](https://ru.wikipedia.org/wiki/AES_(стандарт_шифрования))
