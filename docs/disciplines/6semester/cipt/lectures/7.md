# 7. Гибридные криптосистемы. TLS

## Гибридные криптосистемы

### Определение

**Гибридная (комбинированная) криптосистема** — система шифрования, совмещающая преимущества симметричной и асимметричной криптографии.

### Проблема, которую решает гибридный подход

| Тип шифрования | Достоинства | Недостатки |
|----------------|-------------|-------------|
| Симметричное | Высокая скорость, короткие ключи | Проблема безопасной передачи ключа |
| Асимметричное | Нет проблемы передачи ключа | Низкая производительность, сложные вычисления |

**Решение:** в гибридной системе:
- Данные шифруются **симметричным** алгоритмом (быстро)
- Ключ для симметричного шифрования передаётся с помощью **асимметричного** алгоритма (безопасно)

### Схема работы гибридной криптосистемы

**Этап отправки (Алиса → Боб):**

1. Алиса генерирует случайный **сеансовый ключ** (session key) — 128–512 бит
2. Сообщение шифруется сеансовым ключом (симметричный алгоритм: AES, 3DES и др.)
3. Сеансовый ключ шифруется **открытым ключом Боба** (асимметричный алгоритм: RSA, ECDHE)
4. Алиса отправляет Бобу зашифрованное сообщение + зашифрованный сеансовый ключ

**Этап приёма (Боб):**

1. Боб получает зашифрованное сообщение и зашифрованный сеансовый ключ
2. Боб расшифровывает сеансовый ключ своим **закрытым ключом**
3. Полученным сеансовым ключом Боб расшифровывает сообщение [citation:1][citation:3]

### Где применяются гибридные криптосистемы

- **TLS/SSL** — защита интернет-соединений
- **PGP** — шифрование электронной почты
- **Web-браузеры и Web-серверы**


## TLS (Transport Layer Security)

### Определение

**TLS (Transport Layer Security)** — криптографический протокол, обеспечивающий защищённую передачу данных в компьютерных сетях. Является преемником SSL (Secure Sockets Layer).

### Три ключевых принципа TLS

| Принцип | Описание | Как реализуется |
|---------|----------|-----------------|
| **Шифрование** | Защита от перехвата третьими лицами | Симметричное шифрование (AES 128–256 бит) |
| **Аутентификация** | Подтверждение подлинности сторон | Цифровые сертификаты, центры сертификации (CA) |
| **Целостность** | Обнаружение изменений данных | Коды аутентификации сообщений (MAC) |

### Архитектура TLS

TLS работает на **транспортном уровне** модели OSI (поверх TCP). Двухслойная архитектура:

1. **TLS Record Protocol** — отвечает за шифрование и защиту передаваемых данных
2. **TLS Handshake Protocol** — управляет установлением безопасного соединения

### Процесс установления соединения (TLS Handshake)

| Шаг | Действие |
|-----|----------|
| 1. Client Hello | Клиент отправляет поддерживаемые версии TLS и криптографические алгоритмы |
| 2. Server Hello | Сервер выбирает версию протокола и алгоритмы |
| 3. Server Certificate | Сервер отправляет цифровой сертификат с открытым ключом |
| 4. Key Exchange | Клиент и сервер договариваются об общем секретном ключе (RSA, DH, ECDHE) |
| 5. Finished | Обмен подтверждениями, дальнейшее общение зашифровано |

### Как TLS обеспечивает безопасность

**Конфиденциальность:**
- Используется комбинация симметричного и асимметричного шифрования
- Общий секретный ключ генерируется для одной сессии
- Асимметричное шифрование — только для передачи ключа

**Аутентификация сервера:**
- Клиент проверяет цифровую подпись сертификата
- Проверяется цепочка сертификатов до доверенного корневого CA
- Проверяются даты действия и статус отзыва сертификата

**Свойство прямой секретности (Forward Secrecy):**
- Обеспечивается при использовании ECDHE (Elliptic Curve Diffie-Hellman Ephemeral)
- Даже при компрометации долгосрочного ключа ранее перехваченный трафик остаётся защищённым 

### Где применяется TLS

| Протокол/Приложение | Применение TLS |
|--------------------|----------------|
| **HTTPS** | Защита веб-трафика (значок замка в браузере) |
| **SMTP, IMAP, POP3** | Защита электронной почты |
| **VPN** | Защита VPN-соединений |
| **VoIP (SIP)** | Шифрование голосовых звонков |
| **Корпоративные приложения** | Безопасность API и межсервисного взаимодействия |

### Версии TLS

- **TLS 1.0 и 1.1** — устаревшие, небезопасные
- **TLS 1.2** — широко используется
- **TLS 1.3** — современная версия, улучшенная безопасность и производительность [citation:4]

### Связь гибридных криптосистем и TLS

> **TLS является классическим примером гибридной криптосистемы** [citation:3]

> **TLS является классическим примером гибридной криптосистемы**

**TLS как гибридная система:**

- **Асимметричное шифрование (RSA/ECDHE)**
  - Безопасная передача сеансового ключа
  - Аутентификация через сертификаты

- **Симметричное шифрование (AES)**
  - Быстрое шифрование основных данных
  - Используется сеансовый ключ, переданный асимметрично

- **Хэш-функции (MAC)**
  - Контроль целостности данных
