# 7. Гибридные криптосистемы. TLS

## Гибридные криптосистемы

### Определение

**Гибридная (комбинированная) криптосистема** — система шифрования, совмещающая преимущества симметричной и асимметричной криптографии.

### Проблема, которую решает гибридный подход

| Тип шифрования | Достоинства | Недостатки |
|----------------|-------------|-------------|
| Симметричное | Быстрые, короткие ключи | Проблема безопасной передачи ключа |
| Асимметричное | Нет проблемы передачи ключа | Медленные, сложные вычисления |

**Решение:** в гибридной системе:
- Данные шифруются **симметричным** алгоритмом (быстро) - сеансовый ключ
- Сеансовый ключ шифруется с помощью **асимметричного** алгоритма (безопасно)

### Схема работы гибридной криптосистемы

**Этап отправки (Алиса → Боб):**

1. Алиса генерирует случайный **сеансовый ключ** (session key) — 128–512 бит
2. Сообщение шифруется сеансовым ключом (симметричный алгоритм: AES)
3. Сеансовый ключ шифруется **открытым ключом Боба** (асимметричный алгоритм: RSA)
4. Алиса отправляет Бобу зашифрованное сообщение + зашифрованный сеансовый ключ

**Этап приёма (Боб):**

1. Боб получает зашифрованное сообщение и зашифрованный сеансовый ключ
2. Боб расшифровывает сеансовый ключ своим **закрытым ключом**
3. Полученным сеансовым ключом Боб расшифровывает сообщение

### Где применяются гибридные криптосистемы

- **TLS/SSL** — защита интернет-соединений
- **Web-браузеры и Web-серверы**


## TLS (Transport Layer Security)

### Определение

**TLS (Transport Layer Security)** — криптографический протокол, обеспечивающий защищённую передачу данных в компьютерных сетях. Является преемником SSL (Secure Sockets Layer).

### Три ключевых принципа TLS

| Принцип | Описание | Как реализуется |
|---------|----------|-----------------|
| **Шифрование** | Защита от перехвата третьими лицами | Симметричное шифрование (AES 128–256 бит) |
| **Аутентификация** | Подтверждение подлинности сторон | Цифровые сертификаты, центры сертификации (CA) |
| **Целостность** | Обнаружение изменений данных | Коды аутентификации сообщений (MAC) |

### Архитектура TLS

TLS работает на **транспортном уровне** модели OSI (поверх TCP). Двухслойная архитектура:

1. **TLS Record Protocol** — отвечает за шифрование и защиту передаваемых данных
2. **TLS Handshake Protocol** — управляет установлением безопасного соединения

### Как TLS обеспечивает безопасность

- Рукопожатие: клиент и сервер используют асимметричное шифрование для подтверждения подлинности (сертификаты).
- Обмен ключами: по защищенному каналу они согласовывают единый сессионный симметричный ключ.
- Безопасный туннель: вся дальнейшая передача данных шифруется симметричным ключом. Безопасность зависит от секретности ключа.


### TLS как гибридная система

- **Асимметричное шифрование (RSA)**
  - Безопасная передача сеансового ключа
  - Аутентификация через сертификаты

- **Симметричное шифрование (AES)**
  - Быстрое шифрование основных данных
  - Используется сеансовый ключ, переданный асимметрично

- **Хэш-функции (MAC)**
  - Контроль целостности данных
