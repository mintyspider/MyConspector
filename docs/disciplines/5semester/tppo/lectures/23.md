## Организация тестирования
::: details Основная информация
### Рекомендуемый шаблон - V-модель

1. Блочное тестирование - на уровне блоков и маленьких компонент [*(белый ящик)*](19.md#метод-белого-ящика)
2. Интеграционное тестирование - во время сборки компонент [*(черный ящик)*](20.md#метод-черного-ящика)
3. Системное тестирование - программная система в целом [*(черный ящик)*](20.md#метод-черного-ящика)

Для каждой стадии разные методы тестирования.  
Тестирование начинается разработчиками кода, а завершается независимыми тестировщиками.  Автор кода гарантирует свойства и качества кода, который он создал.  
Во время тестирования ПС целиком появляются эксперты.

> Тестирование - неизбежная часть любой профессиональной разработки ПО.

### Критерии завершения тестирования
1. Запланированные тесты проходят без обнаружения ошибок (для несложных систем).
2. Есть оценки критичности ошибок и план отладки.
3. Статистические оценки (% ошибок, % сбоев при выполнении) ошибок меньше заданного порога (для сложных систем).
:::

## Аттестационное тестирование
    тестирование правильности

Используются финальные наборы тестов на основе критериев аттестации из спецификации требований.

### Проверка конфигурации
- документация разработчика
- руководство пользователя
- комплект программного кода (листинги)
- сопровождающая документация (README.md, NEWS)

### $\alpha$ и $\beta$ тестирование

$\alpha$-тестирование выполняется в организации разработчика (в лабораторных условиях). Заказчик приходит и смотрит, как все работает.

$\beta$-тестирование выполняется в организации заказчика с привлечением конечных пользователей (в реальных условиях).

Разработчики должны предусмотреть какие-то тесты для $\alpha$-тестирования. В $\beta$-тестировании привлекается сообщество предметной области.

## Системное тестирование

Выход за рамки участников проекта для привлечения дополнительных участников с целью не просто найти дефекты, а понять <u>причины</u> возникновения ошибок.

Возможные виды тестирования
1. тесты на восстановление (перезагрузка системы)
2. тесты на безопасность (утечка данных)
3. стресс-тесты (экстремальные условия работы)
4. тесты на производительность (скорость работы)
