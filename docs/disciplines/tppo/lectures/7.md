# 7. Процессы управления. Зоны ответственности менеджера

Процессы управления - обязательная часть любой профессиональной разработки.

## Зоны ответственности менеджера

- Предложение по разработке ПО/ПС (менеджер привлекает разработчиков и взаимодействует с заказчиком)
- Подбор персонала (квалификаций разработчиков)
- Планирование работ ([график работ](../lectures/8#график-работ))
- Оценка стоимости, трудозатрат, объема работ
- Контроль хода выполнения работ
- Написание отчетов (численные характеристики, (например, человекочасы), можно объективную информацию получать для дальнейшего контроля проекта)
- Гарантия успешной работы

**Менеджер должен** гарантировать выполнение бюджетных и временных ограничений с учетом бизнес-цели организации относительно разрабатываемого ПО. 

::: details Оффтопик
Организация берет на себя задачу по реализации проекта, выделяются ресурсы, в рамках которых команда должна работать.

> **Предложение по разработке ПС** представляет собой краткий обзор проекта - описание проекта и требований ПС (назначение, предметная область, функции, ограничения и ресурсы), на основе которого разработчики принимают решение об участии в проекте.
:::
